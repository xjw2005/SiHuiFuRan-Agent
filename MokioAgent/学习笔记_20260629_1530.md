## 二、核心调用关系

### 1. `core/agent.py` vs `graph/workflow.py`

```
agent.py（司机）
    ↓ 调用 .stream()
workflow.py（组装好的汽车）
    ↓ 执行节点
nodes.py（发动机/变速箱等零件）
```

- **`agent.py`**：负责流式调度，调用 workflow 的 `.stream()` 方法
- **`workflow.py`**：只负责"接线"——把 nodes 里的函数拼成图，本身不实现逻辑
- **`nodes.py`**：每个节点的具体干活逻辑

### 2. RuntimeState 的作用

`RuntimeState` 是**配置大背包**——把 workspace、审批模式、超时设置等所有配置打包成一个对象，在整个系统中传递：

| 没有 RuntimeState | 有了 RuntimeState |
|------------------|------------------|
| 手里拿 8 样东西出门 | 把所有东西装背包，提着就走 |
| 参数列表老长老长 | 只传一个对象 |

被传到：`build_tools()`、`CheckpointManager`、`TraceRecorder`、所有图节点...

## 三、生成器与流式输出

### 为什么用生成器（yield）？

生成器可以**做一步，吐一步**，用户体验更好：

```python
# 不用生成器（等全跑完才显示）
def run_agent():
    events = []
    events.append("开始规划...")
    # ...等3分钟
    events.append("调用搜索...")
    # ...等5分钟
    return events  # 用户等了8分钟才看到输出

# 用生成器（做一步显示一步）
def stream_agent_events():
    yield "开始规划..."   # 马上显示
    # 跑规划
    yield "调用搜索..."   # 马上显示
    # 跑搜索
    yield ...
```

### 层层嵌套的生成器

```
cli 的 for 循环
    ↑ yield
stream_agent_events()
    ↑ yield
workflow.stream() （LangGraph 内部）
    ↑ yield
节点内部的逻辑
```

每一层都是流式的，所以终端能看到实时输出。

## 四、graph 包内部关系

| 文件 | 角色 | 一句话 |
|------|------|--------|
| `state.py` | 📦 零件库 | 定义 MokioGraphState 状态有哪些字段 |
| `nodes.py` | ⚙️ 零件 | 每个节点函数的实现（planner/verifier/...） |
| `memory.py` | 🧠 附加功能 | 分层记忆读写 |
| `workflow.py` | 🔌 接线图 | 导入节点函数，拼成工作流图 |

**workflow 本身不创造新逻辑，只是把其他文件的节点串起来。**

## 五、workflow.stream() 的 mode 参数

```python
for mode, event in workflow.stream(...):
    if mode == "custom":
        # 自定义事件（节点内部 yield 出来的，比如工具调用）
        yield {"type": "custom_event", "event": event}
    else:
        # mode 是节点名（比如 "planner"）
        # event 是这个节点对 state 的更新
        latest_node = mode
        _merge_graph_update(current_state, event)
        yield {"type": "graph_event", "event": event}
```

| mode 值 | 含义 | 处理方式 |
|---------|------|---------|
| `"custom"` | 自定义事件（工具调用、hand off） | 直接转发 |
| 节点名（如 `"planner"`） | 节点执行完成，state 更新 | 合并到 current_state |

## 六、关键设计思想

1. **分层清晰**：调度层、业务逻辑层、执行层各司其职
2. **流式优先**：所有耗时操作都用生成器/yield，保证用户体验
3. **状态集中**：RuntimeState + MokioGraphState 两个状态对象贯穿全流程
4. **组装分离**：workflow 只负责接线，节点逻辑都在 nodes.py
